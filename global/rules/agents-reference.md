# Agent Reference

## When to Use Agents
- Complex feature requests → planner
- Code just written → code-reviewer
- Architectural decision → architect
- Build failure → build-error-resolver
- Bug investigation → debugger
- Technical research → researcher

## Planning
| Agent | Purpose |
|-------|---------|
| planner | Implementation planning, phase breakdown |
| architect | System design, boundaries, interfaces |
| brainstormer | Trade-off analysis, option evaluation |
| researcher | Technical research, docs, solutions |

## Execution
| Agent | Purpose |
|-------|---------|
| fullstack-developer | Implementation across stack |
| build-error-resolver | Fix build errors, type issues |
| refactor-cleaner | Dead code cleanup, consolidation |
| code-simplifier | Clarity and maintainability refinement |

## Review
| Agent | Purpose |
|-------|---------|
| code-reviewer | Quality, logic, maintainability |
| security-reviewer | Vulnerabilities, auth, secrets |
| database-reviewer | SQL, schema, query optimization |
| python-reviewer | PEP 8, type hints, Python idioms |
| go-reviewer | Idiomatic Go, concurrency patterns |
| go-build-resolver | Go build, vet, linter issues |

## Support
| Agent | Purpose |
|-------|---------|
| debugger | Root cause analysis, regression isolation |
| tester | Test execution and coverage |
| tdd-guide | Test-driven development workflow |
| e2e-runner | End-to-end testing |
| doc-updater | Documentation updates |
| docs-manager | Full documentation management |
| ui-ux-designer | Interface design |
| journal-writer | Incident documentation |
| project-manager | Progress tracking |
| git-manager | Git operations, commits |
| mcp-manager | MCP tool management |
